Create a Mad Lib






Addendum:

Problem: The user needs to input a series of words (noun, verb, adjective, place, and plural noun) to generate a fun Mad Lib story. The task also involves ensuring that all form fields are validated, the generated result is dynamic, and the entire application is visually appealing.

Solution: A form was created to collect user inputs, which were then processed and validated by a backend Express server. CSS styling was applied to ensure the form and result page were visually pleasing and easy to navigate.

Challenges: Ensuring all fields were validated and properly handling static file serving were key challenges, which were resolved by implementing basic validation on the server side and proper file handling for CSS and other assets.

Improvements: The design was made responsive for different screen sizes, and user feedback was provided when inputs were missing or incorrect. Random word generation was added for faster interaction.
